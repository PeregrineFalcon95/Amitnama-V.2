Breadth First Search (BFS) tutorials বাংলায়। Part 2

Learning applications and solving Problems with BFS.

Bfs না পারলে এখান থেকে শিখে ফেলুন!
আমাদের যেহেতু বিএফএস শেখা হয়ে গিয়েছে চলুন এটি ব্যবহার করে প্রবলেম সল্ভ করা শুরু করি।
Bfs সম্পর্কে পরিষ্কার ধারণা পেতে হলে বাইকালারিং প্রবলেম সল্ভ করা উচিত।

বাইকালারিংঃ
Online Judge (Uva) 10004 – Bicoloring প্রবলেমটি ঝটপট করে পরে ফেলুন।
আমাদের একটি গ্রাফ দেওয়া আছে। যেখানে n সংখ্যক নোড আছে আর L সংখ্যক edge আছে।
আমাদের প্রতিটি নোডকে রঙ করতে হবে শুধুমাত্র দুইটি রঙ ব্যবহার করে। এমন ভাবে রঙ করতে হবে যেন সরাসরি connected দুইটি নোড একই রঙ এর না হয়! আমাদের বলতে হবে ইনপুট দেওয়া গ্রাফটিকে দুইটি রঙ ব্যবহার করে রঙ করা যাবে নাকি যাবে না?
নিজে নিজে যথেষ্ট সময় ধরে চেষ্টা করেও না সমাধান করতে পারলেই কেবল নিচের অংশটুকু দেখুন।

সমাধানঃ
বিএফএস ভালোভাবে বুঝে থাকলে খুব সহজেই প্রবলেমটি সল্ভ করতে পারবেন। ধরি আমরা সাদা এবং কালো দুটি রং দিয়ে গ্রাফকে রং করবো।
ধরি আমরা নোড a এর সকল adjacent নোড b চেক করছি। তখন adjacent নোডগুলো দুটি অবস্তায় থাকতে পারে -
1 - নোডটি আগে ভিজিট করা হয়নি।
2 - নোডটি আগেই ভিজিট করা হয়েছে।

প্রথম ক্ষেত্রে আমরা কি করতে পারি?
নোড a এর রং যদি সাদা হয়, তাহলে নোড b কে কালো রং করবো। আর নোড a এর রং যদি কালো হয়, তাহলে নোড b কে সাদা রং করবো।
দ্বিতীয় ক্ষেত্রে কিন্তু আমরা নোড b এর রং বদলাতে পারবো না। কারন ইতিমধ্যেই b কে রং করা হয়েছে। তাহলে আমাদের এখন চেক করতে হবে যে নোড b এবং নোড a কে একই রং করা হয়েছে কিনা! যদি একই রং হয় তাহলে বাইকালারিং সম্ভব নয়। কারন পাশাপাশি দুটি নোড এর রং আলাদা হচ্ছে না।
তাহলে আমাদের বিএফএস এর কোড টা কিভাবে পরিবর্তন করবো? নিজে যথেষ্ট পরিমাণ চেষ্টা করেও না পারলেই তবে নিচের অংশটুকু দেখুন।

আমাদের কিন্তু চাইলে এখন visited array ব্যবহার না করেও বিএফএস করতে পারি। আমাদের দুটি রং করতে হবে। যেসব নোড এখনো ভিজিট করা হয়নি তাদের মার্ক করতে আমরা তৃতীয় আরেকটি রং ব্যবহার করতে পারি। এই কোড এ সেটি হচ্ছে character ‘n’. প্রথমেই আমরা চেক করছি যে নতুন নোড b এর রং ‘n’ কিনা। যদি তা হয় তাহলে নোডটি এখনো ভিজিট করা হয়নি। তারপর আমরা আমাদের প্রয়োজন মতো রং করে নিচ্ছি।
পরের কন্ডিশনে চেক করছি নোডটি ইতিমধ্যেই ভিজিট করা হয়েছে কিনা। যদি ভিজিট করা হয় তাহলে চেক করছি পাশাপাশি দুটি নোড এর কালার সেম কিনা, যদি সেম হয় আমরা false return করে দিচ্ছি।
আমাদের কোড হবে এই রকমঃ 

while( !qu.empty() )
{
    a = qu.front();
    qu.pop();
    int sz = graph [ a ].size();
    for ( int i = 0 ; i < sz; i++ )
    {
        b = graph [ a ] [ i ];
        if ( color [ b ] == 'n' )
        {
            qu.push( b );
            ///Marking the child with opposite color
            if( color [ a ] == 'b' )color [ b ] = 'w';
            else color [ b ] = 'b';
        }
        else if ( color [ b ] == color [ a ] )
        {
            ///Adjacent nodes have same color. So bi-coloring not possible.
            return false;
        }
    }
}

আমাদের bi-coloring সমাধান করা হয়ে গিয়েছে!
যেহেতু আমাদের সমস্যাটি সমাধান করা হয়ে গিয়েছে, সেহেতু আমরা কি এটি নিয়ে চিন্তা করা বন্ধ করে দিবো?
উত্তর: অবশ্যই না। আপনি যদি ভালো ভাবে ভেবে দেখেন আরও কৌতূহলোদ্দীপক কিছু ব্যপার খোঁজে পেতে পারেন।

1 - গ্রাফটিতে যদি odd length এর কোন সাইকেল থাকে তাহলে বাইকালারিং করা সম্ভব নয়।

2 - আমরা কিন্তু রং করার মাধ্যমে নোড এর দুটি সেট তৈরি করছি। একটি সেট এর সব নোড এর রং হবে সাদা। আরেকটি সেট এর সব নোড এর রং হবে কালো। একই সেট এর দুটি নোড কিন্তু কোন edge দিয়ে সংযুক্ত থাকবে না। কারন তাহলে দুটি একই রং এর নোড পাশাপাশি হয়ে যাবে।

3 - আমরা যদি একই component এর নোডগুলোর লেভেল এর দিকে খেয়াল করি। তাহলে দেখতে পারবো সব লেভেল কে আমরা দুটি ভাগে ভাগ করতে পারি। Even এবং Odd. যদি even লেভেল এর কোন নোড এর রং সাদা হয়, তাহলে even লেভেল এর বাকি সব নোড এর রং অবশ্যই সাদা হবে। আর তখন odd লেভেল এর সব নোড এর রং কালো হবে।
আর যদি even লেভেল এর কোন নোড এর রং কালো হয়, তাহলে even লেভেল এর সব নোড এর রং কালো হবে আর odd লেভেল এর সব রং সাদা হবে।

4 - আগের observation ব্যবহার করে কিন্তু আমরা এখন গ্রাফটি রং না করেও solve করতে পারি! একটু চেষ্টা করে দেখুন তো।
আমাদের বিএফএস এর source নোড থেকে যদি দুটি নোড এর লেভেল even হয় এবং সে দুটি নোড adjacent হয় তাহলে কিন্তু বাইকালারিং সম্ভব না। একই ব্যপার odd লেভেল এর জন্যও প্রযোজ্য।

আমরা এই অংশের কোড কিভাবে করতে পারি?
তাহলে আমাদের বিএফএস এর কোড হবে নিম্নরূপ: 

while( !qu.empty() )
{
    a = qu.front();
    qu.pop();
    int sz = graph [ a ].size();
    for ( int i = 0 ; i < sz; i++ )
    {
        b = graph [ a ] [ i ];
        if ( !level [ b ] )
        {
            qu.push( b );
            level [ b ] = level [ a ] + 1;
        }
        else if ( level [ b ] == level [ a ] )return 1;
    }
}

A Node Too Far.
Online Judge (Uva) 336 - A Node Too Far. প্রবলেমটি ঝটপট করে পরে ফেলুন।

প্রবলেম পড়ে ঘাবড়ে যাবেন না। প্রবলেমটি আসলে বিএফএস এর একটি সহজ প্রবলেম।
আমাদের NC সংখ্যক edges দিয়ে গঠিত একটি গ্রাফ দেওয়া আছে। এই গ্রাফে সর্বোচ্চ 30 টি নোড থাকতে পারে। আর কিছু কুয়েরি দেওয়া আছে। প্রতিটি কুয়েরিতে একটি নোড a দেওয়া থাকবে আর একটি ইন্টিজার b দেওয়া থাকবে। আমাদের বলতে হবে নোড a থেকে সর্বোচ্চ b সংখ্যক edge ব্যবহার করেও মোট কতগুলো নোড এ যেতে পারবো না। প্রথমে নিজে চেষ্টা করুন সমাধান করার।

সমাধানঃ
আমাদের কাছে নোড আছে সর্বোচ্চ 30 টি। আমাদের বলা নেই কতগুলো কুয়েরি থাকতে পারে। ধরি, মোট কুয়েরি এর সংখ্যা Q. আমরা প্রতি কুয়েরি তে যদি একবার বিএফএস চালাই তাহলে আমাদের কোডের সর্বোচ্চ কমপ্লেক্সিটি হবেঃ O( Q*30 ).
প্রতি কুয়েরিতে আমরা বিএফএস চালিয়ে দেখবো সর্বোচ্চ b দূরত্বে কতোগুলো নোড আমরা ভিজিট করতে পারি।

খুবই সহজ সমাধান!
প্রবলেমটি কিন্তু এখনো সম্পূর্ণ ভাবে সমাধান করা হয়নি।
একটু ভালোভাবে খেয়াল করলেই দেখতে পারবেন যে বলা আছে নোড হতে পারে সর্বোচ্চ 30 টি। কিন্তু নির্দিষ্ট করে বলা নেই যে নোড এর ভ্যালু সর্বোচ্চ কতো হতে পারে। ধরুন একটি নোড দেওয়া হল যার মান 123456789. আমরা কিন্তু এতো বড় array নিতে পারবো না নোড visited কিনা চেক করার জন্য বা level সেভ করে রাখার জন্য। তাহলে কিভাবে সমাধান করবো?
আমরা আমাদের ভ্যালুগুলো কমপ্রেস করে নিতে পারি!
Array কিভাবে কমপ্রেস করতে হয় সেটি না পারলে এখান থেকে দেখে আসুন।
আশা করি এখন আর এই প্রবলেম এর সমাধান করতে সমস্যা হবে না।

We Ship Cheap.
Online Judge (Uva) 762 - We Ship Cheap. প্রবলেমটি ঝটপট করে পরে ফেলুন।

প্রবলেমটিতে বলা হয়েছে We Ship Cheap নামে একটি শিপিং কোম্পানি প্রোডাক্ট ডেলিভারি দেয়। তারা সবসময় সবচেয়ে সস্তা রাস্তাটাই ব্যবহার করে। আপনাকে একটি source এবং target শহর দেওয়া হবে। আপনাকে source থেকে target এ যাওয়ার Shortest Path টি প্রিন্ট করতে হবে। আর যদি source থেকে target এ যাওয়া না যায় তাহলে বলতে হবে কোন রাস্তা নেই।
চেষ্টা করুন সমাধান করার।

সমাধানঃ
প্রবলেমটি আপাত দৃষ্টিতে সহজ মনে হচ্ছে। আমাদের নোডগুলো স্ট্রিং আকারে দেওয়া আছে। তাই আমরা স্ট্রিং গুলোকে কমপ্রেস করে ইন্টিজার এ নিয়ে নিতে পারি। অথবা স্ট্রিং টাইপের গ্রাফ ব্যবহার করতে পারি। আমরা এখানে কমপ্রেস করে সমাধান করবো।
আমরা source থেকে বিএফএস চালাবো। তারপর চেক করবো target নোড ভিজিট করা হয়েছে কিনা। যদি ভিজিট করা না হয় তাহলে বলে দিবো আমাদের কোন রাস্তা নেই। আর যদি রাস্তা থাকে তাহলে সে রাস্তাটি প্রিন্ট করবো। আমরা ইতিমধ্যেই শিখেছি কিভাবে রাস্তা প্রিন্ট করতে হয়। তাই সেটি করতে সমস্যা হওয়ার কথা নয়।
কিন্তু আমাদের আরও কিছু ব্যপার খেয়াল রাখতে হবে। আমাদের মোট নোড এর সংখ্যা সর্বোচ্চ কতো হতে পারে? বলা হয়েছে নোড হবে দুটি আপারকেস লেটার। তারমানে সর্বোচ্চ 26*26 টি নোড হতে পারে। এই প্রবলেম এ রং আনসার ভারডিক্ট পাওয়ার বড় একটি কারন আছে। প্রবলেম এ কিন্তু বলা নেই যে source এবং target সবসময় ইনপুট দেওয়া গ্রাফে থাকবে। তাই আমাদের চেক করতে হবে যে সোর্স এবং টার্গেট ইনপুট দেওয়া গ্রাফে আছে কিনা। যদি না থাকে তাহলে সেটিকেও কমপ্রেস করে নিতে হবে। আমরা যখন একটি সমস্যার সমাধান করবো তখন সব ধরণের situation কথা মাথায় রাখতে হবে।

একটি Knight এর বর্তমান পজিশন আর তার টার্গেট পজিশন দেওয়া আছে। আমাদের বলতে হবে সে সর্বনিম্ন কতগুলো মুভ এর মাধ্যমে টার্গেট পজিশন এ যেতে পারবে।
দাবা খেলার বোর্ড হচ্ছে 8*8 এর গ্রিড। যেখানে 8 টি row আছে। এবং প্রতি row তে 8 টি column আছে। আমরা আগের প্রবলেম গুলোয় ইনপুট দেওয়া গ্রাফে বিএফএস চালিয়েছি। কিন্তু গ্রিড এ কিভাবে চালাবো? চেষ্টা করে দেখুন তো পারেন কিনা?
